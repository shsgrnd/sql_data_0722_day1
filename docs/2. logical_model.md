# 학사관리시스템 논리 데이터 모델

## 1. 모델 개요

학사관리시스템의 논리 모델은 총 10개의 엔터티로 구성한다.

- PERSON
- STUDENT
- PROFESSOR
- DEPARTMENT
- DEGREE_PROGRAM
- COURSE
- SEMESTER
- CLASS_SECTION
- CLASS_SCHEDULE
- ENROLLMENT

PERSON은 학생과 교수의 공통 인적정보를 관리하는 상위 엔터티이다. STUDENT와 PROFESSOR는 PERSON의 기본키를 상속하여 역할별 고유 정보를 관리한다.

## 2. 식별자 설계 원칙

시스템 내부 관계에 사용하는 식별자와 실제 업무에서 사용하는 번호를 분리한다.

- PERSON.person_id: 시스템 내부 개인 식별자
- STUDENT.student_number: 실제 학번
- PROFESSOR.professor_number: 실제 교수번호
- DEPARTMENT.department_code: 실제 학과 코드
- COURSE.course_code: 실제 과목 코드

student_number와 professor_number는 업무 식별자로서 UNIQUE를 적용하지만, 다른 엔터티의 외래키는 person_id를 참조한다.

## 3. 엔터티 정의

### 3.1 PERSON

학생과 교수에게 공통으로 적용되는 인적정보를 관리한다.

- 기본키: person_id
- 주요 속성: name, email, phone, birth_date, address
- 후보키: email
- 하위 엔터티: STUDENT, PROFESSOR

### 3.2 STUDENT

PERSON 중 학생 역할을 가진 사람의 학적 정보를 관리한다.

- 기본키: person_id
- 외래키:
  - person_id → PERSON.person_id
  - department_id → DEPARTMENT.department_id
  - degree_program_id → DEGREE_PROGRAM.degree_program_id
  - advisor_id → PROFESSOR.person_id
- 업무 식별자: student_number
- 주요 속성: admission_date, academic_status, current_semester
- 지도교수가 없는 경우 advisor_id는 NULL을 허용한다.

### 3.3 PROFESSOR

PERSON 중 교수 역할을 가진 사람의 교원 정보를 관리한다.

- 기본키: person_id
- 외래키:
  - person_id → PERSON.person_id
  - department_id → DEPARTMENT.department_id
- 업무 식별자: professor_number
- 주요 속성: appointment_date, position, office_location

### 3.4 DEPARTMENT

학생, 교수 및 교과목의 소속 단위인 학과를 관리한다.

- 기본키: department_id
- 후보키: department_code, department_name
- 주요 속성: office_location, phone

### 3.5 DEGREE_PROGRAM

학생의 현재 학위과정을 관리한다.

- 기본키: degree_program_id
- 후보키: program_name
- 주요 속성: standard_years, required_credits, description
- 예시 값: 학사, 석사, 박사

### 3.6 COURSE

학교에 등록된 교과목 자체를 관리한다.

- 기본키: course_id
- 외래키: department_id → DEPARTMENT.department_id
- 후보키: course_code
- 주요 속성: course_name, credits, course_level, description

### 3.7 SEMESTER

강의가 운영되는 학년도와 학기를 관리한다.

- 기본키: semester_id
- 주요 속성:
  - academic_year
  - term
  - start_date
  - end_date
  - registration_start
  - registration_end
- 대체키: academic_year + term

### 3.8 CLASS_SECTION

특정 학기에 실제 개설된 교과목의 분반을 관리한다.

- 기본키: class_section_id
- 외래키:
  - course_id → COURSE.course_id
  - semester_id → SEMESTER.semester_id
  - professor_id → PROFESSOR.person_id
- 주요 속성: section_number, capacity
- 대체키: course_id + semester_id + section_number

### 3.9 ENROLLMENT

학생과 개설강의를 연결하고 수강 및 성적 정보를 관리한다.

- 기본키: enrollment_id
- 외래키:
  - student_id → STUDENT.person_id
  - class_section_id → CLASS_SECTION.class_section_id
- 주요 속성:
  - enrollment_date
  - enrollment_status
  - score
  - grade
- 대체키: student_id + class_section_id
- 점수와 등급은 성적 확정 전까지 NULL을 허용한다.

### 3.10 CLASS_SCHEDULE

개설강의의 실제 수업 요일, 시작 시간, 종료 시간 및 강의실을 관리한다.

- 기본키: class_schedule_id
- 외래키:
  - class_section_id → CLASS_SECTION.class_section_id
- 주요 속성:
  - day_of_week
  - start_time
  - end_time
  - classroom
- 대체키:
  - class_section_id + day_of_week + start_time

## 4. 상속 관계

PERSON과 STUDENT, PROFESSOR 사이에는 상위·하위 엔터티 관계를 적용한다.

- PERSON 1 : 0..1 STUDENT
- PERSON 1 : 0..1 PROFESSOR

STUDENT.person_id와 PROFESSOR.person_id는 각 엔터티의 기본키이면서 PERSON.person_id를 참조하는 외래키이다.

이번 설계에서는 한 PERSON이 학생과 교수 역할을 동시에 갖는 경우를 구조적으로 허용한다.

## 5. 엔터티 간 관계

### 학과 관계

- DEPARTMENT 1 : N STUDENT
- DEPARTMENT 1 : N PROFESSOR
- DEPARTMENT 1 : N COURSE

각 학생, 교수 및 교과목은 반드시 하나의 학과에 속한다.

### 학위과정 관계

- DEGREE_PROGRAM 1 : N STUDENT

각 학생은 반드시 하나의 현재 학위과정을 가진다.

### 지도교수 관계

- PROFESSOR 1 : N STUDENT

교수는 여러 학생을 지도할 수 있다. 학생은 최대 한 명의 지도교수를 가질 수 있으며 지도교수가 없는 경우도 허용한다.

### 강의 개설 관계

- COURSE 1 : N CLASS_SECTION
- SEMESTER 1 : N CLASS_SECTION
- PROFESSOR 1 : N CLASS_SECTION

하나의 개설강의는 반드시 하나의 교과목, 학기 및 담당 교수를 참조한다.

### 강의시간 관계

- CLASS_SECTION 1 : N CLASS_SCHEDULE

하나의 개설강의는 한 개 이상의 강의시간을 가질 수 있으며, 각 강의시간은 하나의 개설강의에 속한다.

### 수강 관계

- STUDENT 1 : N ENROLLMENT
- CLASS_SECTION 1 : N ENROLLMENT

학생과 개설강의의 N:M 관계를 ENROLLMENT로 해소한다.

## 6. 정규화 검토

### 제1정규형

- 각 속성에는 하나의 원자값만 저장한다.
- 여러 연락처나 여러 강의를 하나의 속성에 저장하지 않는다.

### 제2정규형

- ENROLLMENT에는 학생명, 교과목명처럼 일부 키에만 종속되는 정보를 저장하지 않는다.
- 수강과 직접 관련된 상태, 점수 및 성적만 저장한다.

### 제3정규형

- STUDENT에는 학과명과 학위과정명을 저장하지 않는다.
- COURSE에는 학과명을 저장하지 않는다.
- 외래키를 통해 DEPARTMENT와 DEGREE_PROGRAM을 참조한다.

## 7. 주요 고유성 제약

- PERSON.email
- STUDENT.student_number
- PROFESSOR.professor_number
- DEPARTMENT.department_code
- DEPARTMENT.department_name
- DEGREE_PROGRAM.program_name
- COURSE.course_code
- SEMESTER의 academic_year + term
- CLASS_SECTION의 course_id + semester_id + section_number
- CLASS_SCHEDULE의 class_section_id + day_of_week + start_time
- ENROLLMENT의 student_id + class_section_id

## 8. 선택적 속성

다음 속성은 값이 아직 정해지지 않았거나 존재하지 않을 수 있으므로 NULL을 허용한다.

- PERSON.phone
- PERSON.address
- STUDENT.advisor_id
- PROFESSOR.office_location
- COURSE.description
- CLASS_SCHEDULE.classroom
- ENROLLMENT.score
- ENROLLMENT.grade
