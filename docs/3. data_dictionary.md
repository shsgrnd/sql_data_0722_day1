# 학사관리시스템 데이터 사전

## 1. 문서 개요

본 문서는 학사관리시스템 논리 데이터 모델에서 사용하는 엔터티와 속성의 의미, 논리 자료형, 식별자, 필수 여부 및 제약조건을 정의한다.

PostgreSQL의 구체적인 자료형과 길이는 물리 데이터 모델링 단계에서 별도로 확정한다.

## 2. 표기 규칙

| 표기 | 의미 |
|---|---|
| PK | 기본키 |
| FK | 외래키 |
| UK | 고유키 또는 대체키 |
| Y | 해당 조건 적용 |
| N | 해당 조건 미적용 |
| NULL 허용 | 값이 정해지지 않거나 존재하지 않을 수 있음 |

---

# 3. PERSON

학생과 교수의 공통 인적정보를 관리하는 상위 엔터티이다.

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| person_id | 개인 식별자 | 시스템 내부에서 개인을 식별하는 값 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| name | 이름 | 개인의 이름 | 문자 | N | N | Y | N | 공백만 입력할 수 없음 |
| email | 이메일 | 개인의 이메일 주소 | 문자 | N | N | Y | Y | 이메일 형식 사용 |
| phone | 연락처 | 개인의 전화번호 | 문자 | N | N | N | N | NULL 허용 |
| birth_date | 생년월일 | 개인의 출생일 | 날짜 | N | N | Y | N | 미래 날짜 입력 불가 |
| address | 주소 | 개인의 거주 주소 | 문자 | N | N | N | N | NULL 허용 |

# 4. DEPARTMENT

학생, 교수 및 교과목이 소속되는 학과 정보를 관리한다.

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| department_id | 학과 식별자 | 시스템 내부 학과 식별자 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| department_code | 학과 코드 | 학과를 나타내는 업무 코드 | 문자 | N | N | Y | Y | 예: CSE, MSE |
| department_name | 학과명 | 학과의 정식 명칭 | 문자 | N | N | Y | Y | 중복 불가 |
| office_location | 학과 사무실 | 학과 사무실 위치 | 문자 | N | N | N | N | NULL 허용 |
| phone | 학과 연락처 | 학과 대표 전화번호 | 문자 | N | N | N | N | NULL 허용 |

# 5. DEGREE_PROGRAM

학생이 소속되는 학사, 석사, 박사 학위과정을 관리한다.

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| degree_program_id | 학위과정 식별자 | 시스템 내부 학위과정 식별자 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| program_name | 학위과정명 | 학사, 석사, 박사 등의 과정명 | 문자 | N | N | Y | Y | 허용값: 학사, 석사, 박사 |
| standard_years | 표준 수업연한 | 해당 학위과정의 표준 이수 연수 | 숫자 | N | N | Y | N | 1 이상의 값 |
| required_credits | 필요 학점 | 학위 취득에 필요한 기준 학점 | 숫자 | N | N | Y | N | 0 이상의 값 |
| description | 과정 설명 | 학위과정에 대한 추가 설명 | 문자 | N | N | N | N | NULL 허용 |

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |

| student_number | 학번 | 학교에서 부여한 학생 업무번호 | 문자 | N | N | Y | Y | 학생별 중복 불가 |
| department_id | 소속 학과 식별자 | 학생이 소속된 학과 | 식별자 | N | Y | Y | N | DEPARTMENT.department_id 참조 |
| degree_program_id | 학위과정 식별자 | 학생의 현재 학위과정 | 식별자 | N | Y | Y | N | DEGREE_PROGRAM.degree_program_id 참조 |

| academic_status | 학적 상태 | 학생의 현재 학적 상태 | 문자 | N | N | Y | N | 재학, 휴학, 수료, 졸업, 제적 |
| current_semester | 현재 이수 학기 | 현재까지 이수하거나 재학 중인 학기 수 | 숫자 | N | N | Y | N | 1 이상의 값 |

강의가 운영되는 학년도와 학기 정보를 관리한다.

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| semester_id | 학기 식별자 | 시스템 내부 학기 식별자 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| academic_year | 학년도 | 강의가 운영되는 연도 | 숫자 | N | N | Y | 조합 고유 | term과 조합하여 중복 불가 |
| term | 학기 구분 | 정규 또는 계절학기 구분 | 문자 | N | N | Y | 조합 고유 | 1학기, 2학기, 여름학기, 겨울학기 |
| start_date | 개강일 | 해당 학기의 수업 시작일 | 날짜 | N | N | Y | N | end_date 이전 |
| end_date | 종강일 | 해당 학기의 수업 종료일 | 날짜 | N | N | Y | N | start_date 이후 |

| registration_end | 수강신청 종료일 | 수강신청 접수 종료일 | 날짜 | N | N | Y | N | registration_start 이후 또는 동일 |

### 복합 고유 제약

- `academic_year + term`

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| class_section_id | 개설강의 식별자 | 시스템 내부 개설강의 식별자 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| course_id | 교과목 식별자 | 개설 대상 교과목 | 식별자 | N | Y | Y | 조합 고유 | COURSE.course_id 참조 |
| semester_id | 학기 식별자 | 강의가 개설되는 학기 | 식별자 | N | Y | Y | 조합 고유 | SEMESTER.semester_id 참조 |
| professor_id | 담당 교수 식별자 | 강의를 담당하는 교수 | 식별자 | N | Y | Y | N | PROFESSOR.person_id 참조 |
| section_number | 분반 번호 | 동일 교과목의 학기별 분반 | 문자 | N | N | Y | 조합 고유 | 예: 01, 02 |
| capacity | 수강 정원 | 수강 가능한 최대 학생 수 | 숫자 | N | N | Y | N | 1 이상의 값 |

### 복합 고유 제약

- `course_id + semester_id + section_number`

# 11. CLASS_SCHEDULE

개설강의가 실제로 진행되는 요일, 시작 시간, 종료 시간 및 강의실을 관리한다.

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| class_schedule_id | 강의시간 식별자 | 시스템 내부 강의시간 식별자 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| class_section_id | 개설강의 식별자 | 강의시간이 속한 개설강의 | 식별자 | N | Y | Y | 조합 고유 | CLASS_SECTION.class_section_id 참조 |
| day_of_week | 요일 | 강의가 진행되는 요일 | 문자 | N | N | Y | 조합 고유 | 월, 화, 수, 목, 금, 토 |
| start_time | 시작 시간 | 강의 시작 시각 | 시간 | N | N | Y | 조합 고유 | end_time 이전 |
| end_time | 종료 시간 | 강의 종료 시각 | 시간 | N | N | Y | N | start_time 이후 |
| classroom | 강의실 | 해당 시간대에 사용하는 강의실 | 문자 | N | N | N | N | 온라인 또는 미정인 경우 NULL 허용 |

## 복합 고유 제약

- `class_section_id + day_of_week + start_time`

하나의 개설강의에 동일한 요일과 시작 시간이 중복 등록되는 것을 방지한다.

# 12. ENROLLMENT

학생과 개설강의를 연결하고 수강 및 성적 정보를 관리한다.

| 속성명 | 한글명 | 설명 | 논리 자료형 | PK | FK | 필수 | 고유 | 참조 및 제약 |
|---|---|---|---|---|---|---|---|---|
| enrollment_id | 수강 식별자 | 시스템 내부 수강정보 식별자 | 식별자 | Y | N | Y | Y | 시스템 내부 생성 |
| student_id | 학생 식별자 | 강의를 수강하는 학생 | 식별자 | N | Y | Y | 조합 고유 | STUDENT.person_id 참조 |
| class_section_id | 개설강의 식별자 | 학생이 수강하는 개설강의 | 식별자 | N | Y | Y | 조합 고유 | CLASS_SECTION.class_section_id 참조 |
| enrollment_date | 수강신청일 | 학생이 수강신청한 날짜 | 날짜 | N | N | Y | N | 해당 학기의 신청 기간 검토 대상 |
| enrollment_status | 수강 상태 | 수강정보의 현재 상태 | 문자 | N | N | Y | N | 수강중, 이수, 취소, 재수강 |
| score | 점수 | 학생이 취득한 점수 | 숫자 | N | N | N | N | 0~100, 성적 확정 전 NULL 허용 |
| grade | 성적 등급 | 점수에 따른 성적 등급 | 문자 | N | N | N | N | A+, A0, B+, B0, C+, C0, D+, D0, F |

### 복합 고유 제약

- `student_id + class_section_id`

동일한 학생은 같은 개설강의에 중복 등록할 수 없다.

# 16. 추후 물리 모델링 시 결정할 항목

다음 항목은 PostgreSQL 물리 데이터 모델링 단계에서 확정한다.

- 각 식별자의 PostgreSQL 자료형
- 문자 속성별 최대 길이
- 내부 식별자의 자동 생성 방식
- 외래키 삭제 및 수정 정책
- CHECK 제약조건의 구체적인 SQL 문법
- DEFAULT 값
- 인덱스 생성 대상
- 날짜 및 시간 속성의 실제 자료형
